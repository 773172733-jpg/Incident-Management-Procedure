# UI_SPEC.md — 事件树视觉与页面规范

## 1. 总体方向

产品气质：
- 微信生态内的精致轻工具；
- 个人长期使用不疲劳；
- 专业但不压迫；
- 信息层级清楚；
- 操作路径短；
- 为未来小团队协作预留空间，但V1保持纯个人体验。

视觉关键词：
`微信小程序原生感`
`清爽白底`
`浅灰背景`
`橙色主操作`
`大留白`
`圆角卡片`
`柔和阴影`
`低饱和辅助色`
`轻量项目管理`
`事件进度可视化`
`温和专业`
`非游戏化`
`非科技黑`
`不繁琐`

设计参考图：
`design/事件树_UI设计稿参考.png`

参考图只用于把握布局、留白、卡片和色彩气质，不得逐像素照抄，也不得复制其中可能不准确的文字。

## 3. 字体层级
- 页面主标题：22px / 600
- 模块标题：18px / 600
- 卡片标题：16px / 600
- 正文：14px / 400
- 辅助文字：12px / 400
- 进度数字：24—30px / 700
- 主按钮：16px / 600

## 4. 首页

顶部：
- 标题“我的事件”
- 副标题“专注目标，让每件事都有进度”
- 避开微信右上角胶囊安全区域

数据卡：
- 新建事件
- 进行中
- 已结束

筛选胶囊：
- 全部
- 进行中
- 无期限
- 已结束

大事件卡：
- 主题图标
- 标题
- 时间文案
- 状态标签
- 进度条
- 完成数 / 总数
- 最近截止任务

三种时间文案：
- `2026.07.01—2026.11.10`
- `持续进行 · 已持续87天`
- `未设置时间`

空状态：
“还没有事件。把一件想做的事拆开，它就会开始变得清晰。”

## 5. 创建/编辑大事件

单页表单：
1. 事件名称
2. 图标和主题色
3. 事件说明
4. 时间模式
5. 对应日期
6. 提醒
7. 初始分组
8. 底部固定保存按钮

时间模式使用三个独立选择卡：
- 不设置时间
- 设置期间
- 持续进行

## 6. 大事件详情

顶部项目卡：
- 图标
- 标题
- 时间
- 状态
- 更多菜单

进度卡：
- 大数字百分比
- 横向进度条
- 已完成数量
- 最近截止任务

分组标签：
- 全部
- 各分组
- 未分组
- 管理分组

优先级筛选：
- 全部
- 核心
- 重要
- 可选

任务条目：
- 优先级标签
- 标题
- 备注摘要
- 时间和提醒
- 右侧完成按钮

完成后：
- 灰色
- 删除线
- 透明度降低
- 显示完成时间
- 默认沉到当前分组底部

## 7. 日历
- 顶部月份切换
- 月历卡
- 有任务日期显示圆点或数量
- 选中日期使用橙色
- 下方显示当天任务
- 逾期只用小范围红色提示

## 8. 动态
顶部：
- 待处理
- 操作记录

时间流：
- 操作图标
- 文案
- 时间
- 所属大事件

布局要能在团队版中加入头像和人员名称，但V1不显示空头像占位。

## 9. 我的
- 用户信息
- 默认提醒
- 完成任务自动下沉
- 已归档
- 回收站
- 通知设置
- 隐私说明
- 关于

## 10. 动效
只使用160—240ms轻动效：
- 勾选缩放
- 进度条平滑变化
- 新任务淡入
- 完成任务淡出并移动
- 底部面板上滑
- 保存按钮 loading

## 11. 禁止项
- 不直接复制其他小程序
- 不做装修验收软件视觉
- 不用大量 emoji
- 不做蓝绿夜景风
- 不做像素风
- 不做复杂背景
- 不把每个区域都加边框
- 不同时使用多种抢眼主色
- V1不出现团队、会员、AI、支付入口
